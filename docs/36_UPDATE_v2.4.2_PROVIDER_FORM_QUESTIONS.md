# อัปเดต v2.4.2 — นำเข้าข้อสอบจาก Blank Result Form และเชื่อม Case Study

## ปัญหาที่แก้

รอบ CAP JE-B 2026 มีเอกสาร 2 บทบาทต่างกัน:

- `CAP-JE-B-2026_JE_BlankResultForm.pdf` มีโจทย์จริง 4 ข้อและตัวเลือกข้อละ 4 ตัวเลือก
- `CAP-JE-B-2026_JE1_KitInstruction.pdf` มี Case Study, ABO/Rh, Antibody screen, DAT, Eluate panel และ Additional studies

รุ่นเดิมอ่านข้อมูลทั้งสองไฟล์เป็นความรู้ทั่วไป แล้วให้ AI แต่งคำถาม Competency ใหม่ จึงเกิดคำถาม ABO/Rh/Screen/AbID/Crossmatch/Antigen typing 6 ข้อที่ไม่ตรงกับโจทย์ของ CAP

## การทำงานใหม่

### 1. อ่านเอกสารแยกตามบทบาท

**Blank Result Form**

ระบบเก็บข้อมูลเป็น `provider_questions` ได้แก่:

- เลขข้อ
- หัวข้อหรือ Dry Challenge
- รหัส Case เช่น JE-14
- ข้อความโจทย์ตามต้นฉบับ
- ชนิดคำถาม
- รหัสตัวเลือก
- ข้อความตัวเลือก
- ตำแหน่งหน้าในเอกสาร

**Kit Instruction**

ระบบเก็บข้อมูลเป็น `case_studies` ได้แก่:

- รหัส Case
- ชื่อ Case
- เนื้อหากรณีศึกษา
- ผลตรวจและข้อมูลสำคัญ
- ตำแหน่งหน้าในเอกสาร

### 2. ไม่แต่งข้อสอบใหม่เมื่อมีข้อสอบต้นฉบับ

เมื่อพบ `provider_questions` ระบบจะ:

1. นำโจทย์จาก Blank Result Form มาใช้ตรง ๆ
2. จัดตัวเลือกเป็น `CAP code │ ข้อความ`
3. เชื่อม Case Study จาก Kit Instruction ด้วย `case_reference`
4. สร้างข้อสอบฉบับร่างในฐานข้อมูลโดยตรง
5. ไม่เรียก OpenAI รอบสร้างคำถามซ้ำ
6. ถ้าระบบคาดว่าจะนำเข้าข้อสอบต้นฉบับ แต่ข้อมูลที่อ่านมาไม่มีโจทย์หรือตัวเลือก ระบบจะหยุดและแจ้งให้ตรวจไฟล์ แทนการแต่งคำถามทั่วไปที่ไม่ตรงเอกสาร

OpenAI ยังถูกใช้ตอนอ่าน PDF ครั้งแรกเท่านั้น

### 3. ไม่กระทบข้อสอบเดิม

- คำถามจากภาพผลดิบยังใช้ระบบ ABO, Rh, Screen, AbID, Crossmatch และ Antigen typing เดิม
- ข้อสอบที่เพิ่มเองไม่ถูกลบ
- ข้อสอบที่เผยแพร่แล้วไม่ถูกลบ
- ระบบแทนที่เฉพาะข้อสอบที่ `generated_by_ai = true`, ยังเป็นฉบับร่าง และยังไม่มีคำตอบเท่านั้น
- ไม่เปลี่ยนโครงสร้างตาราง จึงไม่ต้อง Run SQL

## การใช้ Token

เมื่ออัปเดตเป็น v2.4.2:

- Blank Result Form และ Kit Instruction ต้องอ่านใหม่หนึ่งครั้ง เพราะมีโครงสร้างข้อมูลใหม่
- Raw Result Image, Antigram, Official Evaluation และ Participant Summary ที่อ่านด้วย schema v2.3.2 แล้ว ใช้ Cache เดิมต่อได้
- หลังอ่านเอกสารสำเร็จ การสร้างข้อสอบต้นฉบับไม่เรียก OpenAI ซ้ำ

## วิธีติดตั้ง

### ขั้นที่ 1 — อัปเดต GitHub Pages

อัปโหลดไฟล์ทั้งหมดในชุด v2.4.2 ทับไฟล์เดิม โดยเฉพาะ:

- `index.html`
- `service-worker.js`
- `css/app.css`
- `js/app.js`
- `js/pwa-install.js`

รอ GitHub Pages Deploy สำเร็จ แล้วกด `Ctrl + F5`

### ขั้นที่ 2 — Deploy Edge Function

Deploy Function ชื่อ `generate-competency` ใหม่จาก:

```text
supabase/functions/generate-competency/index.ts
```

ไม่ต้องสร้าง Function ใหม่ และไม่ต้องเปลี่ยน Secrets เดิม

### ขั้นที่ 3 — สร้างข้อสอบ JE-B ใหม่

1. เปิดรอบ `CAP JE-B 2026`
2. ไปที่ `10. การประเมินความสามารถ`
3. กด `สร้างจากเอกสาร`
4. เลือก `สร้างข้อสอบ`
5. ติ๊ก `แทนที่เฉพาะข้อสอบฉบับร่างที่ AI เคยสร้างไว้`
6. ยืนยันเริ่มสร้าง
7. รอระบบอ่าน Blank Result Form และ Kit Instruction ใหม่
8. ตรวจว่ามีข้อสอบ JE-14 จำนวน 4 ข้อ ข้อละ 4 ตัวเลือก
9. QM ตรวจข้อความและตัวเลือกก่อนกด `เผยแพร่ทั้งหมด`
